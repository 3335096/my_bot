# Telegram OpenRouter Agent Bot

Личный Telegram-бот-агент на Python (`aiogram`) + OpenRouter.

Поддержка:
- текстового общения (streaming-ответы);
- голосовых сообщений (транскрибация + streaming-ответ);
- изображений (vision-анализ, streaming);
- документов (PDF и текстовые файлы);
- умного роутинга задач по типу запроса;
- истории диалогов: последние 10, сохраненные, открепление, принудительное удаление.

## Быстрый старт

1. Создайте `.env` из шаблона:

```bash
cp .env.example .env
```

2. Заполните обязательные переменные:
- `TELEGRAM_BOT_TOKEN`
- `OPENROUTER_API_KEY`
- `DATABASE_URL`

3. Установите зависимости и запустите:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -e .
python -m bot.main
```

4. Для максимально надежной транскрибации установите `ffmpeg` локально:

```bash
# Ubuntu/Debian
sudo apt-get update && sudo apt-get install -y ffmpeg
```

## Команды

- `/start` — старт и подсказка
- `/help` — список команд
- `/new` (или `/reset`) — новый диалог
- `/history` — последние 10 диалогов
- `/saved` — сохраненные диалоги

## UX-правила (зафиксировано)

- Routing-бейдж показывается **только в первом ответе новой сессии**.
- Все LLM-ответы приходят **streaming** — текст появляется постепенно.
- Во время загрузки аудио/документа показывается индикатор **typing...**.
- Последние диалоги: строго **10**.
- В списке последних: `Открыть`, `Сохранить` / `Открепить`, `Удалить`.
- В списке сохраненных: `Открыть`, `Открепить`, `Удалить`.
- Лимиты хранения:
  - `RECENT_SESSIONS_LIMIT=10`
  - `SAVED_SESSIONS_LIMIT` (по умолчанию 50)
- При превышении лимита удаляется самый старый диалог из соответствующего списка.
- Кнопка удаления удаляет диалог принудительно сразу.

## Документы

- Поддерживаемые форматы: **PDF** и **текстовые файлы** (txt, md, py, js, csv, json, yaml и др.).
- Опциональная подпись к документу используется как вопрос (без подписи — краткое изложение).
- Лимиты:
  - `DOCUMENT_MAX_FILE_SIZE_MB` (по умолчанию 20 MB),
  - `DOCUMENT_MAX_EXTRACTED_CHARS` (по умолчанию 50 000 символов).
- Во время загрузки и извлечения текста показывается индикатор **typing...**.

## Аудио-пайплайн (надежный режим)

- Бот сначала пытается нормализовать входящее аудио через `ffmpeg` в `wav` (`mono`, `16k`).
- Если первичная транскрибация не прошла и есть валидный исходный формат, бот делает fallback-повтор на исходном файле.
- Если `ffmpeg` недоступен, используется best-effort режим: отправка исходного поддерживаемого формата.
- До транскрибации применяются лимиты:
  - `AUDIO_MAX_DURATION_SECONDS` (по умолчанию 300 сек),
  - `AUDIO_MAX_FILE_SIZE_MB` (по умолчанию 20 MB).

## Retry/backoff OpenRouter

- Для chat/transcription запросов используется retry-политика на временные ошибки:
  - timeout/network exceptions,
  - HTTP `408/409/425/429/500/502/503/504`.
- Параметры:
  - `REQUEST_MAX_RETRIES` (по умолчанию 2),
  - `REQUEST_RETRY_BACKOFF_BASE_SECONDS` (по умолчанию 1),
  - `REQUEST_RETRY_BACKOFF_MAX_SECONDS` (по умолчанию 8).
